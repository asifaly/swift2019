MT 548 Field Specifications

61. Field 12a: Type of Financial Instrument: Classification Type

FORMAT

Option A :4!c/[8c]/30x (Qualifier)(Data Source Scheme)(Instrument Code or Description)
Option C :4!c//6!c (Qualifier)(CFI Code)

PRESENCE

Optional in optional subsequence C1a1A1

QUALIFIER

(Error code(s): T89)

Order M/O Qualifier R/N CR Options Qualifier Description
1 O CLAS N   A or C Classification Type

DEFINITION

This qualified generic field specifies:

CLAS Classification Type Classification type of the financial instrument, for example, ISO Classification of Financial instrument (CFI).

USAGE RULES

When option C is used, CFI code must contain a valid ISO CFI code.

To use the codes associated with the ISITC code for security type, /ASCT/, Data Source Scheme must be ISIT and Instrument Code or Description must contain a valid ISITC security type code.

To use the codes associated with the code for security type defined in the CSDR regulation, SHRS, SOVR, DEBT, SECU, EFTS, UCIT, MMKT, EMAL, OTHR, Data Source Scheme must be REGL and Instrument Code or Description must contain a valid security type code as defined in the CSD Regulation.
